# Domain Docs

This is a single-context repository.

Before exploring or changing domain behavior:

- Read `CONTEXT.md` at the repository root and use its canonical vocabulary.
- Read relevant records under `docs/adr/` when that directory exists.
- If a required domain concept is missing, note the gap for `grill-with-docs` rather than silently inventing competing terminology.
- Surface any conflict with an existing ADR explicitly.
